# Pipeline

三段流水线,每段一种线程,段间有界队列连接。重点不是单个原语,是多角色协调时的关闭协议 + join 拓扑序。

建议阅读顺序:

| # | 文件 | 概念 | 一句话 |
|---|------|------|--------|
| 1 | [Pipeline.cpp](Pipeline.cpp) | 多段队列 + 关闭协议 + join 拓扑序 | Producer→Worker→Sink,源头先 join,被阻塞的消费端最后 join |

主线:单样本起步,先把 3 角色的关闭协议和 join 顺序立稳,后续再扩。

## 关闭顺序为什么是拓扑序

| 规则 | 原因 |
|------|------|
| join 依赖链的源头先收,被阻塞的消费者最后收 | 源头停产并 close 队列,下游才能从 wait 醒来排空退出 |
| 反序 join 会循环等待死锁 | 先 join 消费者,但它在等源头喂数据,源头还没被 join |
| 独立对等节点之间 join 顺序自由 | 无依赖边,先后无所谓 |
| 链上有环 = 设计错误 | 用单向 stop 信号 / 超时 / 重构成 DAG / C++20 `std::jthread`+`std::stop_token` 破环 |

后续可加 `std::jthread` / `stop_token` 协作取消样本,免手排 join 序。
