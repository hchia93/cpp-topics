## Try / Wait 模板

`Try*` 看一眼立即返回 (非阻塞), `Wait` 在 `Try*` 外套 spin 循环直到成功. Test_07 / Test_08 / Test_09 / Test_10 全用此套路.

```cpp
bool TryAcquire() {                  // 看一眼, 立即返回
    if (/* condition not met */) return false;
    // CAS / fetch_sub / load 等单步原子动作
    return true;
}

void Wait() {                        // 套 spin
    while (!TryAcquire()) {
        std::this_thread::yield();   // 或 sleep / pause
    }
}
```

好处: 把"看一眼"和"等到为止"解耦, 测试可分别覆盖, busy/idle 策略可替换.

## release/acquire 决策表

| 形状 | 谁通知 (release) | 谁被通知 (acquire) | 通知靠什么变量 |
|------|------------------|--------------------|-----------------|
| 1→N | 老板挂出"开门"牌子 | 排队的人看见牌子才进 | 那块牌子 |
| N→1 | 每个乘客上车告诉司机"我到了" | 司机等所有人到齐才开车 | 已上车的人数 |
| N↔N peer 互斥 | 上一个用浴室的人推门出来 | 下一个排队的进门反锁 | 门是否被占 |
| M↔N permit | 还车的人把车放回车架 | 租车的人从车架取车 | 车架上剩几辆 |

必须命中同一个 atomic 变量, 屏障保护方向: release 保护在它**之前**的写, acquire 保护在它**之后**的读.

## spin 实现速查

| 写法 | 适用时机 | 画面 |
|------|---------|------|
| hot spin (`while (!try) {}`) | 等几纳秒, 持锁极短 | 眼睛盯着红绿灯不眨眼, 绿了立刻冲 |
| cooperative spin (`yield` / `sleep_for(us)` / `_mm_pause`) | 等几微秒到几毫秒 | 看一眼信号, 转头和旁边人聊两句, 再回头看 |
| block (`condition_variable::wait` / `atomic::wait`) | 等几毫秒以上 | 找椅子坐下, 信号变了有人来拍你肩膀叫醒 |

## 卡住时回答三个问题

- **谁通知, 谁被通知**: 你这次写代码, 是在告诉别人事情发生了 (release), 还是在等别人告诉你 (acquire)? 选错一端就读到旧值, 像信号还没挂出来你就走进店里
- **通知之前你做了什么, 对方要看到吗**: 比如老板挂"开门"牌子之前, 菜单要先写好; 乘客对司机说"我到了"之前, 行李要先放好. release 屏障保护这些"之前的写"
- **这事一次性还是反复**: 火箭发射前点火, 只点一次 (one-shot, latch / flag); 共享单车则是每天租来还去 (reusable, semaphore / cv). 用错形状第二轮卡死
